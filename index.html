How It Works

문제 풀이 방식

01

문제를 고릅니다

난이도와 SQL 개념을 보고 사건, 보안 로그, 운영 데이터 문제를 선택합니다.

02

데이터 지도를 봅니다

ERD와 샘플 행으로 테이블 관계를 파악하고, 직접 조사 방향을 정합니다.

03

자유롭게 쿼리합니다

JOIN, GROUP BY, 서브쿼리 등 원하는 SQLite 문법으로 단서를 좁힙니다.

04

결론을 제출합니다

정답 SQL이 아니라 증거와 최종 판단이 맞는지로 풀이를 완료합니다.

For Developers

GitHub 기반 학습 프로필

GitHub 계정으로 로그인하면 나중에 풀이 기록, 랭킹, 공개 프로필을 개발자 정체성과 자연스럽게 연결할 수 있습니다. 프로그래머스식 운영 요소를 가져오되, StudieSQL은 데이터 탐색력과 추론력을 중심에 둡니다.

준비됨: 문제 보드, 진행 지표, GitHub OAuth 다음: 유저별 풀이 로그 저장 이후: 공개 프로필, 랭킹, 해설 비교

Learning Path

SQL 문법을 외우기보다, 필요한 질문을 만드는 연습

StudieSQL의 학습은 강의실 순서가 아니라 조사 흐름을 따라갑니다. 테이블을 보고, 조건을 좁히고, 관계를 연결하고, 집계로 이상 징후를 찾는 방식으로 단계가 열립니다.

Curriculum

현재 학습 흐름

Lv.1

테이블에서 단서 찾기

SELECT, WHERE, ORDER BY로 사건 기록에서 필요한 행을 찾아냅니다.

Lv.3

흩어진 기록 연결하기

JOIN, LIKE, 서브쿼리로 접속 로그와 계정 정보를 이어 봅니다.

Lv.4

패턴과 이상 징후 보기

GROUP BY, HAVING으로 반복, 누락, 과도한 요청 같은 신호를 찾습니다.

Principle

정답 쿼리보다 사고 과정

같은 답에 도달하는 SQL은 여러 개일 수 있습니다. 그래서 StudieSQL은 정해진 쿼리 한 줄보다 결과에서 어떤 증거를 발견했는지, 최종 보고서가 맞는지를 중심으로 진행률을 계산합니다.

먼저 데이터 지도에서 테이블 관계를 확인합니다. 작게 조회하고, 결과를 보고 다음 조건을 정합니다. 마지막 제출 전에는 근거가 충분한지 확인합니다.

Progress

풀이 기록은 지금까지의 조사 습관을 보여줍니다

아직은 브라우저에 저장되는 개인 기록입니다. 앞으로 로그인 계정과 연결해 공개 프로필, 랭킹, 풀이 로그 비교로 확장할 수 있습니다.
